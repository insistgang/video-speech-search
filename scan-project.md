## 扫描流程

### 第一步：项目结构扫描
1. 读取项目根目录结构（2层深度）
2. 识别关键文件：README、配置文件、模型文件、数据集目录、训练脚本、论文草稿
3. 统计代码规模：文件数量、代码行数、主要语言占比

### 第二步：代码健康度检查
1. 检查是否有 requirements.txt / pyproject.toml / package.json
2. 检查 .gitignore 是否存在且合理
3. 检查是否有未提交的改动（git status）
4. 检查最近10条 git log，分析开发节奏
5. 检查是否有 TODO/FIXME/HACK 注释，统计数量和位置

### 第三步：深度分析（按项目类型适配）

**如果是深度学习/CV项目：**
- 扫描模型定义文件，列出网络架构（backbone、neck、head）
- 扫描训练配置，提取关键超参数（lr、batch_size、epochs、optimizer）
- 检查数据集目录：类别数、样本量、标注格式（YOLO/COCO/VOC）
- 扫描 results/runs 目录，找到最佳模型权重和对应指标（mAP、loss等）
- 检查是否有消融实验记录

**如果是 Web/App 项目：**
- 扫描技术栈（框架、数据库、API）
- 检查路由/页面结构
- 检查环境配置文件
- 检查测试覆盖率

**如果是论文/文档项目：**
- 扫描章节结构和完成度
- 检查参考文献数量
- 检查图表数量和位置

### 第四步：问题诊断
1. 找出潜在的技术债：过大的文件、重复代码、废弃文件
2. 找出阻塞项：缺失的依赖、断裂的导入、空的占位模块
3. 找出安全隐患：硬编码的密钥、暴露的配置

### 第五步：生成报告

输出一份 `PROJECT_STATUS.md` 到项目根目录，包含以下模块：

