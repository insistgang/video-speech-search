# 视频画面内容检索平台操作手册

适用项目目录：`E:\understand_mov`  
适用时间：2026-03-31

## 1. 产品简介

视频画面内容检索平台用于处理无语音或低语音的录屏视频，通过抽帧、画面理解、文本索引和关键词检索，帮助快速定位视频中出现的 AI 工具、网页、代码、操作步骤和可疑行为。

当前版本的核心能力：

- 导入单个本地视频
- 批量导入文件夹中的多个视频
- 使用 `ffmpeg` 按固定时间间隔抽取关键帧
- 使用 Kimi 或兼容模型分析关键帧内容
- 将分析结果写入 SQLite FTS5 全文索引
- 在前端页面中按关键词、视频、时间范围、AI 工具命中情况进行检索
- 查看命中画面的结构化详情和视频定位时间点
- 建立关键词集并对全库批量扫描
- 查看任务列表、历史任务和失败原因

当前版本不包含的能力：

- 不支持直接导入在线视频 URL
- 不支持向量检索
- 不支持服务重启后自动恢复未完成任务

## 2. 系统架构概览

项目主要由两部分组成：

- 后端：FastAPI + SQLite + FFmpeg + Kimi 分析链路
- 前端：React + Vite 中文管理界面

关键目录说明：

- `backend/`：后端接口、数据库、抽帧、分析、检索逻辑
- `frontend/`：前端页面与接口调用
- `data/db/`：SQLite 数据库文件
- `data/frames/`：抽取出的关键帧图片
- `data/logs/`：前后端运行日志
- `docs/`：文档目录
- `作弊视频/`：本地测试视频样本目录

## 3. 环境要求

建议环境：

- Windows + PowerShell
- Python 3.12 或以上
- Node.js 18 或以上
- `ffmpeg` 与 `ffprobe`
- 可用的 Kimi 或 Moonshot API 配置

建议先确认以下命令可执行：

```powershell
python --version
node --version
ffmpeg -version
ffprobe -version
```

如果 `ffmpeg` 或 `ffprobe` 不可用，视频导入后将无法抽帧。

## 5. 环境变量说明

项目根目录下提供了示例文件：

- `.env.example`

关键变量说明：

- `MOONSHOT_API_KEY`：模型服务 API Key
- `MOONSHOT_BASE_URL`：模型服务基础地址
- `MODEL_NAME`：模型名称
- `VISION_ANALYZER_MODE`：分析模式
- `KIMI_CLI_COMMAND`：本地 `kimi` CLI 命令名
- `FRAME_INTERVAL`：抽帧间隔，单位秒
- `API_CONCURRENCY`：并发分析数
- `API_MAX_RETRIES`：分析失败时的最大重试次数
- `DB_PATH`：SQLite 数据库文件路径
- `DATA_DIR`：数据根目录
- `FRAMES_DIR`：关键帧输出目录
- `VITE_API_BASE_URL`：前端调用后端的接口基础地址

## 6. 推荐配置方式

### 6.1 本地演示模式

适用于只验证界面流程，不调用真实模型。

```powershell
$env:VISION_ANALYZER_MODE="mock"
$env:DB_PATH="data/db/search.db"
$env:DATA_DIR="data"
$env:FRAMES_DIR="data/frames"
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
```

说明：

- 会生成模拟分析结果
- 不消耗模型额度
- 适合前端联调和流程验证

### 6.2 Kimi Coding Key 多模态模式

适用于使用 Kimi Code CLI 和 `coding/v1` 风格的 Key。

```powershell
$env:MOONSHOT_API_KEY="你的Key"
$env:MOONSHOT_BASE_URL="https://api.kimi.com/coding/v1"
$env:MODEL_NAME="kimi-for-coding"
$env:VISION_ANALYZER_MODE="kimi_cli"
$env:KIMI_CLI_COMMAND="kimi"
$env:FRAME_INTERVAL="10"
$env:API_CONCURRENCY="3"
$env:API_MAX_RETRIES="5"
$env:DB_PATH="data/db/search.db"
$env:DATA_DIR="data"
$env:FRAMES_DIR="data/frames"
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
```

说明：

- 当前项目已经支持通过本地 `kimi` CLI 读取图片并返回结构化 JSON
- 建议在真实视频回填时将 `FRAME_INTERVAL` 设置为 `10`
- 如果希望更密集抽帧，可改回 `3`
- `API_MAX_RETRIES` 建议不低于 `3`

### 6.3 Moonshot 直连模式

适用于可以直接调用兼容 OpenAI `chat.completions` 的图片理解接口。

```powershell
$env:MOONSHOT_API_KEY="你的Key"
$env:MOONSHOT_BASE_URL="https://api.moonshot.cn/v1"
$env:MODEL_NAME="kimi-k2.5"
$env:VISION_ANALYZER_MODE="live"
$env:FRAME_INTERVAL="3"
$env:API_CONCURRENCY="3"
$env:API_MAX_RETRIES="3"
$env:DB_PATH="data/db/search.db"
$env:DATA_DIR="data"
$env:FRAMES_DIR="data/frames"
$env:VITE_API_BASE_URL="http://127.0.0.1:8000/api"
```

## 8. 首次使用流程

建议按以下顺序操作：

1. 启动后端
2. 启动前端
3. 进入“导入”页导入视频或文件夹
4. 进入“任务”页确认任务完成
5. 进入“检索”页输入关键词进行搜索
6. 点击命中结果进入详情页查看画面与视频定位
7. 如需批量检查多个关键词，进入“词库”页创建关键词集并扫描

## 9. 页面使用说明

### 9.1 检索页

页面用途：

- 搜索已经完成索引的视频内容
- 按关键词、视频、时间范围、AI 工具识别状态过滤

支持的过滤项：

- 关键词
- 视频
- AI 工具检测
- 起始时间
- 结束时间

使用方法：

1. 在检索框输入关键词
2. 可选设置视频和时间范围
3. 点击“开始检索”
4. 浏览命中结果卡片
5. 点击结果进入详情页

检索说明：

- 查询不能为空
- 输入文件名、平台名、工具名、页面文字、代码关键字都可命中
- 系统会把查询拆词后送入 SQLite FTS5，因此带标点符号的输入也可以搜索

常见有效关键词示例：

- `Copilot`
- `Bedrock`
- `DeepAI`
- `DynamoDB`
- `AWS`
- `在IDE侧使用AI.mp4`

### 9.2 导入页

页面用途：

- 导入单个视频
- 导入文件夹内所有支持的视频文件
- 查看当前视频库

支持的视频扩展名：

- `.mp4`
- `.mov`
- `.avi`
- `.mkv`
- `.webm`
- `.m4v`

单视频导入步骤：

1. 进入“导入”页
2. 在“视频路径”中输入完整本地路径
3. 点击“导入并排队”

批量导入步骤：

1. 在“文件夹路径”中输入目录路径
2. 点击“批量导入”

说明：

- 文件夹导入会递归扫描子目录
- 每个视频会自动入库并创建处理任务
- 导入后可在页面底部“视频库”中查看状态

### 9.3 任务页

页面用途：

- 查看视频处理进度
- 查看历史任务
- 查看任务错误原因

当前任务页行为：

- 默认只显示每个视频的最新任务
- 点击“显示历史任务”可查看历史重试记录

字段说明：

- `状态`：任务状态 / 视频状态
- `进度`：任务完成百分比
- `帧数`：已处理帧数 / 总帧数
- `Token 用量`：模型调用统计
- `错误`：失败原因或提示信息

说明：

- 如果你看到同一个视频有旧失败记录和新成功记录，说明系统曾做过重试
- 默认视图已经隐藏旧任务，只保留最新结果

### 9.4 词库页

页面用途：

- 创建关键词集
- 批量扫描视频库
- 聚合多个关键词的命中结果

创建关键词集步骤：

1. 输入名称
2. 输入分类
3. 输入关键词，多个关键词用英文逗号或逐项拆分后再粘贴
4. 点击“保存关键词集”

扫描步骤：

1. 在已有关键词集中点击“立即扫描”
2. 系统会对该关键词集中的每个关键词逐个检索
3. 合并相同帧的命中结果
4. 在下方展示命中列表

### 9.5 结果详情页

页面用途：

- 查看命中关键帧
- 查看对应视频并自动定位到命中时间
- 查看结构化分析结果

详情页包含的信息：

- 视频文件信息
- 命中时间点
- 视频播放器
- 命中关键帧预览
- 同一视频的时间线关键帧跳转
- 结构化分析字段

结构化分析字段说明：

- `应用 / 网站`
- `AI 工具识别`
- `页面链接`
- `风险标签`
- `画面摘要`
- `操作描述`
- `可见文字`
- `代码摘要`

## 11. 常用接口一览

后端基础地址：

- `http://127.0.0.1:8000/api`

核心接口：

- `GET /api/health`：健康检查
- `GET /api/videos`：列出已导入视频
- `POST /api/videos/import`：导入单个视频
- `POST /api/videos/import-folder`：导入文件夹
- `GET /api/tasks`：列出任务
- `POST /api/tasks/{task_id}/retry`：重试指定任务
- `POST /api/search`：执行检索
- `GET /api/search/results/{frame_id}`：查看命中详情
- `GET /api/frames/{frame_id}/image`：获取关键帧图片
- `GET /api/videos/{video_id}/file`：获取视频文件
- `GET /api/keywords`：列出关键词集
- `POST /api/keywords`：创建关键词集
- `DELETE /api/keywords/{keyword_set_id}`：删除关键词集
- `POST /api/keywords/{keyword_set_id}/scan`：按关键词集扫描

## 12. 常见问题与处理方法

### 12.1 页面提示 `错误：422 Unprocessable Entity`

含义：

- 请求已经到后端
- 但请求参数不符合接口校验规则

在检索页最常见原因：

- 搜索关键词为空

处理方法：

1. 在搜索框中输入至少 1 个字符
2. 再点击“开始检索”

接口校验规则中，`query` 至少需要 1 个字符。

### 12.2 页面打不开

检查顺序：

1. 确认后端是否启动
2. 访问 `http://127.0.0.1:8000/api/health`
3. 确认前端终端输出中的实际地址
4. 如果 `5173` 被占用，改用 `5174` 或终端实际端口

### 12.3 导入视频后任务长期不动

常见原因：

- 后端未启动
- `ffmpeg` / `ffprobe` 不可用
- 模型服务配置错误
- 数据库路径切换到了另一份空库

检查方法：

```powershell
ffmpeg -version
ffprobe -version
```

并查看：

- `data/logs/backend.err.log`

### 12.4 任务失败，错误里提到 JSON 不完整

含义：

- 模型返回了非严格 JSON 或中间被截断

当前系统已做改进：

- `kimi_cli` 模式支持失败重试
- 建议将 `API_MAX_RETRIES` 设置为 `3` 到 `5`

### 12.5 任务页里出现旧失败任务

说明：

- 历史任务记录不会自动删除
- 如果该视频后来又重试成功，任务页默认只展示最新任务
- 如展开历史任务，则可能看到旧失败记录

### 12.6 搜索结果为空

可能原因：

- 视频还没处理完成
- 当前使用的是另一份数据库
- 搜索词过窄
- 过滤条件过多

建议排查：

1. 先在“任务”页确认视频状态为“已完成”
2. 先用宽泛关键词试搜，例如 `AWS`、`AI`、`Copilot`
3. 去掉时间范围和 AI 检测过滤

## 13. 常用运维操作

### 13.1 切换数据库

```powershell
$env:DB_PATH="data/db/real_kimi_test_v2.db"
```

切换后需要重启后端。

### 13.2 调整抽帧间隔

```powershell
$env:FRAME_INTERVAL="10"
```

说明：

- 值越小，抽帧越密，识别更细，但处理更慢
- 值越大，处理更快，但可能漏掉短时画面变化

推荐值：

- 快速批量回填：`10`
- 细粒度检测：`3`

### 13.3 重试失败任务

可以在 Swagger 页面直接调用：

- `POST /api/tasks/{task_id}/retry`

也可以用 PowerShell：

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:8000/api/tasks/7/retry -Method Post
```

### 13.4 运行测试

后端：

```powershell
cd E:\understand_mov
pytest -q
```

前端：

```powershell
cd E:\understand_mov\frontend
npm run test
npm run build
```

## 14. 推荐日常操作流程

### 场景一：导入新一批录屏视频并检索

1. 启动后端和前端
2. 进入“导入”页批量导入文件夹
3. 进入“任务”页等待任务完成
4. 进入“检索”页搜索 AI 工具名或页面关键词
5. 进入详情页复核命中结果

### 场景二：批量筛查是否使用特定 AI 工具

1. 进入“词库”页
2. 创建关键词集，例如 `AI 工具排查`
3. 输入关键词，例如 `ChatGPT, Claude, Copilot, Bedrock, DeepSeek, Kimi`
4. 点击“立即扫描”
5. 查看聚合命中结果

### 场景三：重新分析旧视频

1. 确认新的环境变量配置
2. 重启后端
3. 对旧任务调用重试接口
4. 在“任务”页查看新任务是否完成

## 15. 当前已知限制

- 任务队列为内存队列，后端重启会中断未完成任务
- 不支持自动清理旧任务历史
- 不支持从页面直接点击“重试任务”
- 不支持空查询返回全部结果
- 前端开发服务器端口可能因占用而变化

## 16. 建议

建议在正式使用时固定以下策略：

- 使用独立数据库文件，不要与临时调试库混用
- 保持 `API_MAX_RETRIES >= 3`
- 真实视频分析优先使用 `FRAME_INTERVAL=10` 做首轮扫描
- 对重点视频再使用更小间隔重新处理
- 每次切换数据库或模型配置后重启后端
