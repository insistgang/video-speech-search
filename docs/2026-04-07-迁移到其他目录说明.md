# 迁移到其他目录说明

## 结论

当前项目在迁移到其他目录或其他机器时，确实存在两个典型风险：

1. `ffmpeg` / `ffprobe` 如果不在目标环境的 `PATH` 中，导入和抽帧会失败。
2. 已导入数据中保存的是绝对路径，直接搬动项目目录后，旧记录可能仍然指向原目录。

本次代码已经补上两项兼容处理：

1. 新增 `.env` 配置项 `FFMPEG_COMMAND` 和 `FFPROBE_COMMAND`，不需要再去源码里写死本机绝对路径。
2. 导入路径会自动清理首尾空格和外层引号，避免把 `"D:\video\demo.mp4"` 误解析成项目内相对路径。

## 迁移前要知道的事

### 1. 现在不要再改源码里的 FFmpeg 路径

请改 `.env`，不要改 Python 文件。

可选方案：

- 方案 A：把 `ffmpeg` 和 `ffprobe` 加入系统 `PATH`
- 方案 B：在 `.env` 中显式配置可执行文件绝对路径

示例：

```env
FFMPEG_COMMAND=C:\ffmpeg\bin\ffmpeg.exe
FFPROBE_COMMAND=C:\ffmpeg\bin\ffprobe.exe
```

如果已经加入 `PATH`，保持默认值即可：

```env
FFMPEG_COMMAND=ffmpeg
FFPROBE_COMMAND=ffprobe
```

### 2. 旧数据库里的路径是绝对路径

当前数据库中至少有两类字段会保存绝对路径：

- `video_assets.filepath`
- `video_frames.image_path`

这意味着：

- 如果你把项目从 `E:\understand_mov` 搬到别的目录，旧的帧图路径大概率会失效
- 如果数据库里还留着旧机器或旧目录的视频记录，检索结果可能还能搜到，但点开详情时视频或帧无法加载

## 推荐迁移方式

### 方案一：干净迁移，最稳

适合继续开发、演示、重新导入视频。

步骤：

1. 把项目代码复制到新目录。
2. 复制 `.env.example` 为 `.env`，按新环境修改配置。
3. 配置 `FFMPEG_COMMAND` / `FFPROBE_COMMAND`，或者确保它们在 `PATH` 中。
4. 删除旧目录下的 `data/db` 和 `data/frames`，或者不要把旧 `data` 一起搬过去。
5. 在新目录重新安装依赖并启动项目。
6. 重新导入视频，重新生成抽帧和分析结果。

这是最推荐的方式，因为不会残留旧绝对路径。

### 方案二：连旧数据一起迁移

只适合你明确知道自己在做什么。

要求：

1. 原视频文件仍然存在。
2. 数据库中保存的绝对路径在新环境下仍然有效，或者你自己额外写脚本批量改库。

如果你只是把项目目录换了位置，但数据库里还保留旧的 `image_path` / `filepath`，前端详情页仍然可能打不开。

如果没有必须保留的历史数据，不建议走这条路。

## 新目录最小配置清单

至少确认下面这些项：

```env
VISION_PROVIDER=zhipu
VISION_API_KEY=你的密钥
VISION_BASE_URL=https://open.bigmodel.cn/api/paas/v4
MODEL_NAME=glm-4.6v-flash
VISION_ANALYZER_MODE=live

FFMPEG_COMMAND=ffmpeg
FFPROBE_COMMAND=ffprobe

FRAME_INTERVAL=5
FRAME_MAX_WIDTH=1280
FRAME_JPEG_QUALITY=5
API_CONCURRENCY=1
API_MAX_RETRIES=3

DB_PATH=data/db/search.db
DATA_DIR=data
FRAMES_DIR=data/frames
VITE_API_BASE_URL=http://127.0.0.1:8000/api
```

如果目标机器没有配置 `PATH`，把上面两项改成绝对路径即可。

## 迁移后验证

先验证 FFmpeg：

```powershell
ffmpeg -version
ffprobe -version
```

如果你使用的是 `.env` 中的绝对路径配置，也可以直接在 PowerShell 里验证：

```powershell
& "C:\ffmpeg\bin\ffmpeg.exe" -version
& "C:\ffmpeg\bin\ffprobe.exe" -version
```

再启动后端和前端：

```powershell
uvicorn backend.main:app --reload
```

```powershell
cd frontend
npm install
npm run dev
```

最后做一次完整检查：

1. 导入一个测试视频。
2. 观察任务状态是否从 `pending` 进入 `running` 再到 `completed`。
3. 确认 `data/frames/video_xxxx/` 下实际生成了 `jpg`。
4. 在检索页搜索一个明显关键词，确认结果详情页能打开视频和帧图。

## 常见故障

### 提示找不到 `ffprobe`

原因：

- 目标机器没有安装 FFmpeg
- 安装了，但没有加入 `PATH`
- `.env` 里没有配置 `FFPROBE_COMMAND`

处理：

- 安装 FFmpeg，或
- 在 `.env` 中补上 `FFPROBE_COMMAND`

### 导入成功，但抽帧失败

原因：

- `ffprobe` 能找到，但 `ffmpeg` 不能找到
- `FFMPEG_COMMAND` 配置错误

处理：

- 检查 `FFMPEG_COMMAND`
- 直接运行对应命令确认可执行文件是否存在

### 检索能搜到结果，但详情打不开

原因：

- 数据库里是旧目录的绝对路径
- 视频文件或帧图文件已经不在原位置

处理：

- 清空 `data/db` 和 `data/frames` 后重新导入
- 或自己写脚本批量更新数据库绝对路径

## 本次变更对应的实际效果

现在项目迁移时：

- 不需要再手改源码里的 `ffmpeg` / `ffprobe` 路径
- 允许通过 `.env` 在不同机器配置不同的可执行文件位置
- 用户把带双引号的路径粘贴进导入框时，不会再被错误解析为项目内相对路径

剩余仍需人工处理的部分只有一类：历史数据库里的绝对路径数据。
