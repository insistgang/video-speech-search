# understand_mov_v2 新电脑与服务器启动说明手册

本文档用于在新电脑或服务器上首次拉起 `understand_mov_v2` 项目，覆盖两种场景：

- 本地开发启动：适合新电脑联调、开发、调试
- 服务器部署启动：适合 Linux 服务器使用 Docker Compose 持久化运行

## 1. 项目说明

项目是一个“视频画面内容检索平台”：

- 后端：FastAPI + SQLite + Alembic
- 前端：React + Vite
- 视频处理：FFmpeg / ffprobe
- 任务队列：SQLite 持久化队列
- 默认数据库：`data/db/search.db`
- 默认帧目录：`data/frames`

## 2. 新电脑启动

### 2.1 准备环境

Windows 或 macOS / Linux 本地开发建议准备以下环境：

- Python 3.12
- Node.js 20
- npm 10+
- FFmpeg
- Git

可先验证：

```powershell
python --version
node --version
npm --version
ffmpeg -version
ffprobe -version
```

### 2.2 获取代码

```powershell
git clone <你的仓库地址> understand_mov_v2
cd understand_mov_v2
```

### 2.3 配置环境变量

复制模板文件：

```powershell
Copy-Item .env.example .env
```

至少检查并修改这些关键项：

```dotenv
VISION_PROVIDER=zhipu
VISION_API_KEY=你的模型密钥
VISION_BASE_URL=https://open.bigmodel.cn/api/paas/v4
MODEL_NAME=glm-4.6v-flashx
VISION_ANALYZER_MODE=live

API_KEY=your-secure-random-key-here
VITE_API_KEY=your-secure-random-key-here
VITE_API_BASE_URL=http://127.0.0.1:8000/api

DB_PATH=data/db/search.db
DATA_DIR=data
FRAMES_DIR=data/frames
ALLOWED_VIDEO_DIRS=.;.\作弊视频
```

说明：

- `API_KEY` 和 `VITE_API_KEY` 必须保持一致
- 如果没有真实模型密钥，想先调 UI，可以改成 `VISION_ANALYZER_MODE=mock`
- Windows 下 `ALLOWED_VIDEO_DIRS` 用 `;` 分隔
- Linux/macOS 下 `ALLOWED_VIDEO_DIRS` 用 `:` 分隔

### 2.4 安装后端依赖

建议先创建虚拟环境：

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt -r requirements-dev.txt
```

### 2.5 安装前端依赖

```powershell
cd frontend
npm install
cd ..
```

### 2.6 初始化数据库

项目已接入 Alembic。首次启动前执行迁移：

```powershell
python -m alembic upgrade head
```

如果迁移成功，数据库结构会创建到 `DB_PATH` 指向的位置。

### 2.7 启动后端

```powershell
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```

验证健康检查：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/api/health -UseBasicParsing
```

### 2.8 启动前端

新开一个终端：

```powershell
cd frontend
npm run dev -- --host 127.0.0.1 --port 3000
```

如果 `3000` 被占用，Vite 会自动切到 `3001`、`3002` 等其它端口。

### 2.9 本地访问地址

- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:8000`
- 健康检查：`http://127.0.0.1:8000/api/health`

### 2.10 本地验证命令

```powershell
python -m pytest -q
cd frontend
npm run test
npm run build
cd ..
```

## 3. 服务器启动

### 3.1 服务器准备

建议 Linux 服务器准备：

- Docker
- Docker Compose Plugin
- 至少 2 CPU / 4 GB RAM
- 已安装 Git

验证环境：

```bash
docker --version
docker compose version
git --version
```

### 3.2 拉取代码

```bash
git clone <你的仓库地址> understand_mov_v2
cd understand_mov_v2
```

### 3.3 配置 `.env`

```bash
cp .env.example .env
```

服务器部署最少确认这些字段：

```dotenv
VISION_PROVIDER=zhipu
VISION_API_KEY=你的真实模型密钥
VISION_BASE_URL=https://open.bigmodel.cn/api/paas/v4
MODEL_NAME=glm-4.6v-flashx
VISION_ANALYZER_MODE=live

API_KEY=your-secure-random-key-here
VITE_API_KEY=your-secure-random-key-here
VITE_API_BASE_URL=/api

VIDEO_IMPORT_PATH=/data/videos
ALLOWED_VIDEO_DIRS=/app/videos

DB_PATH=data/db/search.db
DATA_DIR=data
FRAMES_DIR=data/frames
```

说明：

- `VIDEO_IMPORT_PATH` 是宿主机目录，会挂载进容器里的 `/app/videos`
- 前端容器会在构建期读取 `VITE_API_KEY`
- 后端容器启动时会自动执行 `python -m alembic upgrade head`

### 3.4 准备视频目录

例如服务器上把待导入视频放在：

```bash
mkdir -p /data/videos
```

那么 `.env` 中应设置：

```dotenv
VIDEO_IMPORT_PATH=/data/videos
```

容器内实际访问路径会是：

```text
/app/videos
```

通过界面或接口导入时，应填写容器内路径，例如：

```text
/app/videos/demo.mp4
```

### 3.5 构建并启动

```bash
docker compose build
docker compose up -d --force-recreate
```

### 3.6 查看服务状态

```bash
docker compose ps
docker compose logs -f backend
docker compose logs -f frontend
```

### 3.7 服务器访问地址

当前 `docker-compose.yml` 端口映射为：

- 前端：`8080:8080`
- 后端：`8000:8000`

因此默认访问：

- 前端：`http://服务器IP:8080`
- 后端健康检查：`http://服务器IP:8000/api/health`

### 3.8 服务器验证命令

```bash
curl http://127.0.0.1:8000/api/health
curl http://127.0.0.1:8080
```

### 3.9 常用运维命令

重建并重启：

```bash
docker compose up -d --build --force-recreate
```

停止服务：

```bash
docker compose down
```

查看实时日志：

```bash
docker compose logs -f
```

只看后端日志：

```bash
docker compose logs -f backend
```

## 4. 常见问题

### 4.1 前端打开后提示 401

通常是 `API_KEY` 和 `VITE_API_KEY` 不一致，或者前端重新构建前没有更新 `.env`。

处理方式：

- 确认 `.env` 中这两个值一致
- 重新执行 `docker compose build`
- 再执行 `docker compose up -d --force-recreate`

### 4.2 Docker 启动后导入不到视频

通常是宿主机目录没有挂载对，或者导入时填成了宿主机路径而不是容器路径。

正确做法：

- 宿主机目录：`/data/videos`
- `.env`：`VIDEO_IMPORT_PATH=/data/videos`
- 界面中填写：`/app/videos/xxx.mp4`

### 4.3 后端启动报数据库结构问题

如果是本地启动，先手动执行：

```powershell
python -m alembic upgrade head
```

如果是 Docker 启动，容器会在启动命令里自动执行迁移。可通过后端日志确认：

```bash
docker compose logs backend
```

### 4.4 本地没有模型密钥，只想验证流程

把 `.env` 改成：

```dotenv
VISION_ANALYZER_MODE=mock
```

然后重新启动后端即可。

### 4.5 FFmpeg 命令找不到

说明系统没有装 FFmpeg，或者不在 PATH。

处理方式：

- 安装 FFmpeg
- 确认 `ffmpeg -version` 和 `ffprobe -version` 能执行
- 必要时在 `.env` 中设置：

```dotenv
FFMPEG_COMMAND=/usr/bin/ffmpeg
FFPROBE_COMMAND=/usr/bin/ffprobe
```

## 5. 推荐启动顺序

### 新电脑

1. 安装 Python / Node / FFmpeg
2. 复制 `.env.example` 到 `.env`
3. 安装 Python 依赖和前端依赖
4. 执行 `python -m alembic upgrade head`
5. 启动后端
6. 启动前端
7. 跑测试确认

### 服务器

1. 安装 Docker
2. 配置 `.env`
3. 准备宿主机视频目录
4. 执行 `docker compose build`
5. 执行 `docker compose up -d --force-recreate`
6. 检查 `docker compose ps` 和日志

## 6. 最终验证

本地或服务器都可以使用以下检查项：

- `/api/health` 返回 `{"status":"ok"}`
- 前端首页能正常打开
- 可以看到任务页、搜索页、导入页
- 可以成功导入一个测试视频
- 任务处理结束后可以搜索到结果
